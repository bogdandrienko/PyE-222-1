tcp/ip - 

1) SOAP(2%) - легаси, вызывает процедуры прям на сервере.

2) GRPS(2%) - сверхсовременный, нужен где высокая нагрузка.

3) HTTP/REST-api/REST/RESTfull(90%) - request-response(~0-20s)
(поиск домена-> поиск ip -> создание тунеля -> подготовка данных)
[40% - 60%]
(вспомогательные операции -> основные операции)

4) WEBSOCKET(5%)  - длительное соединение(1 час)
* уведомления
* онлайн чаты
* частые обновления данных(бегущая строка с криптовалютой)
* Обновление данных графиков, новостей, всяких цифер типа коэффициентов

Разговор со знакомым:
* REST - после каждого предложения, снова здороваться.
* WEBSOCKET - один раз здороваться и до окончания соединения.




Сервер + Клиенты
1) Запускает веб-сокет на сервере и "слушает" все соединения.
2) Браузер пытается открыть веб-сокет(соединение) с сервером.
3) Со стороны браузера(клиента): можно получать сообщения, либо отправлять.
4) Со стороны сервера: он либо получает сообщения, либо отправляет.
